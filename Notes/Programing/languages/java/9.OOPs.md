> [!Key rule]
> A non-static inner class belongs to an instance of the outer class, so you can't do new InnerClass() inside a static method (like main). Fix: add static to the inner class — static class Student { — to make it a static nested class. That's the right call anyway when the inner class doesn't access outer-class instance fields.

```java
public class demo {
  static class Student{
    String sname;
    int age;
    int rollno;
    String college;
  }
  public static void main(String[] args) {
    Student s1; // reference to object
    s1=new Student(); // heap allocation
    s1.sname="meow";
    s1.age=21;
    s1.rollno=123;

    Student s2=new Student();
    s2.sname="kitty";
    s2.age=22;
    s2.rollno=124;
    System.out.println(s1.sname+" "+s1.age+" "+s1.rollno); // meow 21 123
    System.out.println(s2.sname+" "+s2.age+" "+s2.rollno); // kitty 22 124
  }
}
```
This class are non-primitive data types -> can have use-defined data type
here, `new` is dynamic memory allocation at run-time

###### default value
- for primitive(stack memory) variable there, is no default value. gives error if try to use un-initialized
- for instance variable have default value(string -> null, int -> 0, boolean -> false) because of optimization for heap storage
###### need to do something when object is created like give values to avoid default value. Rules of constructor
- it is special method with same name as that of class
- has no return type not even void
- automatically call when object is created
- It can also be overloaded
- Always have default constructor -> `constructor{}`
```java
public class demo {
  public static void main(String[] args) {
    Student s1=new Student("Sachin",22,10);
    s1.print();   // Sachin 22 10
    Student s2=new Student("Rahul");
    s2.print();   // Rahul 0 0
    Student s3=new Student();
    s3.print();   // null 0 0
  }
}
  class Student{
    String name;
    int age;
    int rollno;
    Student(String n,int a,int r){
      this.name=n;
      this.age=a;
      this.rollno=r;
    }
    Student(String n){
      this.name=n;
    }
    Student(){
    }
    void print(){
      System.out.println(name+" "+age+" "+rollno);
    }
  }
```
###### `this` stores reference to current object -> used if parameter name is same as property name
## chaining to avoid construction overloading
```java
  class Student{
    String name;
    int age;
    int rollno;
    String college;
    Student(String n,int a,int r,String c){ // parameterized constructor
      name=n;
      age=a;
      rollno=r;
      college=c;
    }
    Student(String n,int a,int r){ // final constructor
      this(n,a,r,"Unknown");
    }
    Student(String n,int a){
      this(n,a,0);
    }
    Student(String n){
      this(n,0);
    }
    Student(){
      this("Unknown");
    }
    void print(){
      System.out.println(name+" "+age+" "+rollno+" "+college);
    }
  }
```
###### Construction can't be called manually -> can be only called when object created and when use `this()` keyword
###### if not get memory when run-time -> give runtime exception
