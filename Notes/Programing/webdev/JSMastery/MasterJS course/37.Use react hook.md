### Problem
###### Have a server action of `hasVoted` to check if user has up-voted down-voted or not voted in a question.
###### calling this server action on question details page will cause blockage on rendering content so need to use something so can find votes in background and not block content
###### solution use hook from reach 
## Use hook
 [use – React](https://react.dev/reference/react/use)
 ###### TL;DR --> It is used to get resource by resolving a promise instead of waiting for it to resolve(pass promise from server to client component). It works well with suspense and loading

###### Use !! to convert to boolean in TS
```TSX
export async function hasSavedQuestion(
  params: CollectionBaseParams,
): Promise<ActionResponse<{ saved: boolean }>> {
  const validationResult = await action({
    params,
    schema: CollectionBasedSchema,
    authorize: true,
  });

  if (validationResult instanceof Error) {
    return handleError(validationResult) as ErrorResponse;
  }
  const { questionId } = validationResult.params!;
  const userId = validationResult.session?.user?.id;
  try {
    const collection = await Collection.findOne({
      author: userId,
      questions: questionId,
    });
      return { success: true, data: { saved: !!collection } }; // !! used to convert to boolean
  } catch (error) {
    return handleError(error) as ErrorResponse;
  }
}
```
