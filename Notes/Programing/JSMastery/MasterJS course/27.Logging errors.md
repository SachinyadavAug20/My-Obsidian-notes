When your app’s running, **logging is key to keeping track of what’s happening,** catching **errors**, and debugging. But... not all loggers are created equal. So, what are our options?

1. **Console**
    
    First up, we've got `console.log`. Easy, built-in, and familiar.
    
    Great ****. for quick logs, but... it's super basic. No structured logging, no extra featurest’s not really built for larger apps.
    
2. **Winston**
    
    Next, there's Winston, **a popular logger with cool features like customization formats and different log levels (like `error`, `warn`, `info`).**
    
    Plus, you can send logs to multiple places: console, files, and even databases. But all that **flexibility comes with a bit of a performance hit**.
    
3. **Bunyan**
    
    Then we’ve got Bunyan. It’s **JSON-based**, so the logs are easy to parse and analyze, especially for server-side use.
    Bit older and limited compare to others
    
1. Pino :Super-fast, all natural JSON(super handy for tracking and monitoring), and is lightweight with minimal overhead.
We will use Pino.
## Why Pino?

Long story short: Pino is fast, structured, and super efficient for server-side applications. So, if you want a logger that's reliable and doesn't slow your app down, Pino's the way to go.

Logging may seem simple, but picking the right tool can make a huge difference in performance and clarity. So, let's give Pino a shot and see how it fits with your Next.js app!

```TS
import pino from "pino";

const isEdge = process.env.NEXT_RUNTIME === "edge";
const isProduction = process.env.NEXT_RUNTIME === "production";

const logger = pino({
  level: process.env.LOG_LEVEL || "info",
  transport:
    !isEdge && !isProduction
      ? {
          target: "pino-pretty",
          options: {
            colorize: true,
            ignore: "pid,hostname",
            translateTime: "SYS:standard",
          },
        }
      : undefined,
  formatters: {
    level: (label) => ({ level: label.toUpperCase() }),
  },
  timestamp: pino.stdTimeFunctions.isoTime,
});

export default logger;

```

will get this error : ![[Pasted image 20260429122527.png]] because the Pino and Pino-pretty cause error when used in nextJS --> solve(use server external packages) by going to next.config.json ==>
```TS
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  /* config options here */
  serverExternalPackages:['pino','pino-pretty'], // HERE
  images:{
    remotePatterns:[
      {
        protocol: 'https',
        hostname: 'avatars.githubusercontent.com',
        port:"",
      }
    ]
  }
};

export default nextConfig;

```

> [!Explanation]
> As nextJS tries to bundle all packages together, but sometimes some items(like `pino`,`pino-pretty`) need to be separated from main bundle, because they are too big or can't be squished with other packages
> =>So, use `serverExternalPackages` it tells nextJS don't put these specific items in main bundle, keep the separate.
