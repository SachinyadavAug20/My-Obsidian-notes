###### flow of code :- how code is code is executed(top to bottom execution) -> some keywords in java can break this flow and bring custom control of code flow.
###### There are 3 types of flow of control
- Selection
	- if
	- if-else
	- switch
- iteration
	- for 
	- for-each
	- while
- Jumps
block of code is represented by `{   //code   }`
## Selections
###### here, expression should be boolean `if(expression){}` 
```java
if(exp){
	// some code
}
```
if else if ladder
```java
if(exp1){
	// code block 1
} else if(exp2) {
	// code block 2
} else if(exp3){
	// code block 3
} else {
	//code block default
}
```
other example
```java
    int i=5;
    if(i==5){
      System.out.println("i is 5");
    }else{
      System.out.println("i is not 5");
    }
```
similar to C/C++ we have  `{` becomes optional when code is 1 line
```java
    int i=5;
    if(i==5)
      System.out.println("i is 5");
    else
      System.out.println("i is not 5");
```
nested if same as in C/C++ ==> avoid more nesting use logical operators
```java
    int age=10;
    if(age>80) System.out.println("You are old");
    else if(age>60) System.out.println("You are getting old");
    else if(age>40) System.out.println("You are mid");
    else if(age>20) System.out.println("You are young");
    else if(age>18) System.out.println("You are adult");
    else System.out.println("you are child");
```
###### switch statement are like if-else-ladder but quicker(optimized)
```java
    switch (age) {
      case 18:
        System.out.println("18");
        break;
      case 19:
        System.out.println("19");
        break;
      case 20:
        System.out.println("20");
        break;
      default:
        System.out.println("IDK");
        break;
    }
    // Importance of break
        int age=19;
    // switch on age
    switch(age){
      case 18: System.out.println("age is 18");
      case 19: System.out.println("age is 19");
      case 20: System.out.println("age is 20");
    }
    /*
     *  age is 19
     *  age is 20
     */
```
###### it is optimize for checking/comparing  --> have limited use cases `switch(exp) here, exp can be byte,short,int,Enumeration,char` , after JDK-14 can `exp in switch can be string,` enhancement

| swith statement                                                                                                                                          | if-else-if-ladder                                |
| -------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------ |
| 1.only equality check                                                                                                                                    | 1.can have any boolean check                     |
| 2.more efficient (because compare together and single evaluation makes jump table(array) which are made when it is optimize is possible) less evaluation | 2.less efficient (evaluate any and many boolean) |
| 3. jump table made when value are close not sparse.for sparse value it will make lookup-switch and does binary search both are O(logn)                   | 3.no such thing                                  |
###### can nest switch in each other.
```java
    int i=10,j=20;
    // nested switch statement
    switch(i){
      case 10:
        switch(j){
          case 20:
            System.out.println("i & j are 10 & 20");
          case 30:
            System.out.println("i & j are 10 & 30");
        }
      case 20:
        System.out.println("i is 20");
    }
```
