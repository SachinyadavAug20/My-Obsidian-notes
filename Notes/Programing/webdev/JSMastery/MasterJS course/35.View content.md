# NextJS After function
[Functions: after | Next.js](https://nextjs.org/docs/app/api-reference/functions/after)
###### Used in server component, schedule work after some response or render is finished
```TS
import { after } from 'next/server'
// Custom logging function
import { log } from '@/app/utils'
 
export default function Layout({ children }: { children: React.ReactNode }) {
  after(() => {
    // Execute after the layout is rendered and sent to the user
  })
  return <>{children}</>
}
```
![[Pasted image 20260511203019.png]]
Points about after function
- It runs asynchronously after the response has been sent to the client.
- It  does not affect the response time experienced by client
- It;s useful for task that don't need to block the response, such as logging, analytics(views) or cleanup operations
- It can access information about the response but can't modify it.
This flow allows for efficient request handling while still enabling important post-response task to be performed.
