### Types of state management
###### types -> Local and global

---
### Local state management
###### It involves managing the data within a single component or module of an app.This local state is specific to that component and is not directly accessible or modifiable by other parts of the app unless explicitly passed down as props or through the other means.
e.g Using useState
``` JSX
"use client"
import React,{useState} from "react";
export default const Counter=()=>{

	const [count,setCount]=useState(0);
	const increment=()=>{
		setCount(count+1);
	}
}
//make a button so increase and decrese count if other components need it share by props drilling
```
### Global state management
###### It is used for the data which is to be accessed and modified by multiple component across the app.--> It is stored at centralized location from where we can access and modify it.
e.g Using context API
```TSX
import React,{createContext,useContext,useState} form "react";

// Create
const CounterContext=createContext();

// provider
const CounterProvider=({children})=>{
	const [count,setCount]=useState(0)
	const increment=()=>{
		setCount(count+1);
	}
	return (
		<CounterContext value={(count,increment)}>
			{children}
		</CounterContext>
	)	
}
export default CounterProvider;
// wrap anything where we need to provide context wrap in CounterProvider an will get it
```

###### There are many Global state management libraries
- Redux : Redux helps organize your app's data in one place using action and reducers
- Zustand : Zustand is a simple way to handle your app's data using react hooks
- Context API : Here we did it (no external library needed)
- Recoil : make managing data in react easier with atoms an derived states
- MobX : make it easy to watch and update your data without extra setup
###### There no "best" one.each solved specific problem efficiently.Choose the best for your app.
