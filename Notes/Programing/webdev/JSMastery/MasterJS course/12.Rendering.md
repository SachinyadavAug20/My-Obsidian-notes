## Rendering strategies
###### There are 3 strategies SSG,ISG,SSR
#### Static Site generation (SSG)
###### Famous, made(html css and JS) on build time =>it requires no user-interaction.The generate static page file can be hosted on content delivery system(CDN) and then served to client as-is.(no processing at runtime)
###### result in cached and reused on subsequent request leading fast and less server load => minimal processing result in high performance 
###### BUT,It is not updated content and if update is done need a re-build for the entire website.There is no interactivity. Examples ![[Pasted image 20260412182924.png|568]]
#### Incremental Site generation (ISG)
###### It allows to update content without re-building entire site
###### The on-demand generation allows to generate specific page on-demand(after set no of second) => certain part of the page will be rendered at runtime.most of pages render at build time while other is generated only when needed(i.e run time)
